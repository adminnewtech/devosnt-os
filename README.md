- **١١ مشروع** يمثلون خط الإنتاج من الفكرة إلى التسليم

## How the factory works (English)

```
01 App Requests  ─►  02 Product Specs  ─►  03 UX UI  ─┐
                                                      ├─► 05 Parallel Build ─► 06 QA & Security ─► 07 Deploy
                                       04 Architecture ┘                                              │
                                                                                                      ▼
                                              09 Research  ◄────────  08 Skill Library  ◄─── Post-build extraction
```

Every IDEA enters via `01 - App Requests` and exits as a live deployed app. The CEO orchestrator routes; the squads execute in parallel; the autopilots guarantee forward progress.

### Self-improvement loop
1. **Factory Pulse v3** — every 15 min: routes, promotes, unblocks, dispatches.
2. **Factory Health Daily Audit** — every day: runtime, WIP, costs, SLOs.
3. **Skill Improvement Weekly Loop** — every week: mines retry logs, proposes new skills + prompt fixes.
4. **cross-project-pattern-extractor** — fires on every closed project.
5. **reusable-template-extractor** — promotes any 3x-repeated workflow into a skill.

## How this repo stays alive

| Trigger | Action |
|---|---|
| Push to `main` | Validate JSON + render markdown + commit any drift |
| Daily 04:00 UTC | Multica → JSON snapshot → re-render → commit |
| Weekly Monday 06:00 UTC | Skill improvement digest from Multica autopilot results |
| Manual `workflow_dispatch` | Force full re-sync |

The repo is **derived state** — Multica is the source of truth. The repo gives Multica a durable, diff-able, GitHub-native history.
